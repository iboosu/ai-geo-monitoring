## 1. 平台是什么

AI GEO Monitoring 是一个生成式引擎优化（Generative Engine Optimization，GEO）监测平台。

平台通过向豆包、DeepSeek 等 AI 平台提交真实用户问题，采集 AI 回答，并分析：

- 品牌是否被提及；
- 品牌在回答中的出现次数和出现顺序；
- AI 是否明确推荐品牌；
- 品牌相对竞品的可见度和声量占比；
- AI 回答引用了哪些网站、域名和页面；
- 是否引用品牌官网或竞品官网；
- 回答对品牌呈现正面、中性还是负面态度；
- 不同 Prompt、平台和时间周期的表现差异；
- 哪些内容和信源值得优先优化。

### 1.1 平台适合解决的问题

- 用户问“某类产品哪个好”时，AI 是否会提到你的品牌？
- AI 更常推荐你还是竞品？
- 品牌在豆包和 DeepSeek 上的表现是否不同？
- 哪些问题中品牌完全缺席？
- AI 回答主要引用哪些网站？
- 竞品被引用、但品牌官网没有被引用的原因是什么？
- 品牌在 AI 回答中的情绪倾向是否变差？
- 最近 7 天、30 天、90 天的 GEO 表现是否改善？

### 1.2 当前重点支持的平台

- 豆包（Doubao）
- DeepSeek

项目数据结构保留了其他平台字段，但当前用户工作台的主要监测范围是豆包和 DeepSeek。

### 1.3 使用平台前的必要条件

1. 系统管理员已经启动前端、后端和数据库。
2. 管理员已配置至少一个可用的 AI 平台服务凭证。
3. 豆包除了 API Key，还必须开通对应模型或配置可用的推理接入点 ID。
4. 用户已创建品牌项目。
5. 项目中至少有一条已启用、平台匹配的 Prompt。

## 2. 角色与权限

### 2.1 普通用户

普通用户主要使用 GEO 项目工作台，可以：

- 创建和维护自己的品牌项目；
- 配置品牌别名、官网、行业和核心关键词；
- 维护竞品；
- 创建、生成、编辑、启用和停用 Prompt；
- 运行单条 Prompt 或整个项目；
- 查看项目看板、来源分析和历史详情；
- 生成周期报告；
- 配置项目告警；
- 查看个人会员等级和每日检测配额。

普通用户只能访问自己的项目和数据。

### 2.2 管理员

管理员除可使用普通用户功能外，还可以：

- 查看全站统计；
- 查看所有用户的检测记录；
- 创建、禁用、删除用户；
- 重置用户密码；
- 调整用户角色和会员等级；
- 配置豆包和 DeepSeek；
- 测试平台真实连接；
- 修改各会员等级的每日检测配额；
- 设置新用户默认会员等级；
- 发布系统通知；
- 修改站点 SEO 设置；
- 查看后端健康状态。

### 2.3 账号状态

- `active`：账号正常，可登录和使用。
- `disabled`：账号被禁用，无法正常登录。

## 3. 10 分钟快速上手

### 第一步：登录

打开：

```text
http://localhost:3001/login
```

使用用户名或邮箱登录。

### 第二步：创建品牌项目

进入“品牌项目”并点击“新建项目”，至少填写：

- 品牌名称；
- 监测平台；
- 建议同时填写官网、别名、行业和核心关键词。

### 第三步：添加竞品

在项目列表中点击“竞品”，添加 2 至 5 个最直接的竞品，并尽量填写：

- 竞品名称；
- 常用别名；
- 官网。

竞品配置越准确，SOV、竞品排名和来源归属越有参考价值。

### 第四步：创建 Prompt

进入“Prompt 库”，选择刚创建的项目。

可以：

- 手工新建 Prompt；
- 在“生成 Prompt 建议”中输入生成重点并批量生成。

建议首批准备 20 至 50 条 Prompt。

### 第五步：运行

在 Prompt 列表中点击单条“运行”，或回到品牌项目点击“运行分析”。

一次运行通常按以下方式计算：

```text
运行单元数 = 选中的可运行 Prompt 数 × 每条 Prompt 的有效监测平台数
```

例如：10 条 Prompt 同时监测豆包和 DeepSeek，最多形成 20 个平台检测单元，并消耗相应配额。

### 第六步：查看结果

- 单条答案：Prompt 库 → 历史；
- 汇总表现：项目看板；
- 引用网站：来源分析；
- 固化周期结果：报告中心；
- 自动监测风险：告警设置。

## 4. 账号注册与登录

### 4.1 注册普通用户

打开：

```text
http://localhost:3001/register
```

填写：

- 用户名：至少 3 个字符；
- 邮箱：必须是有效邮箱格式；
- 密码：至少 6 位；
- 确认密码；
- 图形验证码。

注册成功后，系统会尝试自动登录并进入 GEO 工作台。

### 4.2 登录

登录框支持：

- 用户名；
- 邮箱。

管理员登录成功后进入管理员后台，普通用户进入 GEO 工作台。

### 4.3 登录失败

常见原因：

- 用户名或密码错误；
- 账号被管理员禁用；
- 后端未运行；
- 前端无法连接后端；
- 登录请求触发频率限制。

连续多次失败时，请等待一段时间再试，避免触发登录限流。

## 5. 品牌项目

品牌项目是所有 GEO 数据的顶层容器。Prompt、竞品、运行记录、指标、来源、报告和告警都属于某个品牌项目。

### 5.1 新建项目

进入：

```text
GEO 工作台 → 品牌项目 → 新建项目
```

#### 品牌名称

填写品牌最正式、最常用的名称。

示例：

```text
Goodie AI
```

品牌名称会参与提及识别，因此不要填写公司全称和产品名的混合长句。

#### 品牌别名

填写市场上可能出现的其他写法，例如：

- 中文名；
- 英文名；
- 缩写；
- 不同空格、连字符写法；
- 常用产品品牌名。

系统会对品牌名称和别名去重后进行匹配。

#### 官网

用于识别“自有来源”。建议填写完整 URL：

```text
https://example.com
```

如果不填写官网，系统无法可靠判断某个引用是否属于品牌自有来源。

#### 行业

行业用于帮助生成 Prompt 建议和理解项目背景。

示例：

- AI 营销；
- SaaS；
- 新能源汽车；
- 食品饮料；
- 消费电子。

#### 品牌核心关键词

建议填写：

- 品牌产品名；
- 重点型号；
- 品牌专属服务名称；
- 具有品牌辨识度的产品词。

系统不会把所有通用行业词都算作品牌提及。包含字母和数字的型号类词更适合被识别为品牌产品词。

#### 监测平台

选择豆包、DeepSeek 或两者。

项目平台决定：

- Prompt 可选择的平台范围；
- 项目运行时实际调用的平台；
- 看板、来源和报告统计的平台范围；
- 告警比较的平台范围。

#### 自动监测

开启后，系统按照“每日监测时间”自动运行项目中可运行的 Prompt。

注意：

- 后端服务必须持续运行；
- AI 平台必须可用；
- 用户必须有足够配额；
- Prompt 必须处于启用状态；
- Prompt 平台必须与项目平台匹配。

### 5.2 管理竞品

项目列表 → “竞品”。

每个竞品可以填写：

- 名称；
- 别名；
- 官网。

竞品信息用于：

- 统计竞品提及次数；
- 判断品牌与竞品在答案中的出现顺序；
- 计算竞品可见度得分；
- 计算品牌 SOV；
- 识别竞品来源；
- 生成“竞品压制”和“竞品来源缺口”机会。

建议不要把经销商、行业媒体或完全不同品类的公司作为竞品。

### 5.3 运行整个项目

点击“运行分析”后，系统会运行项目中所有满足条件的 Prompt：

- Prompt 已启用；
- Prompt 至少有一个平台与项目平台匹配；
- 对应平台已配置服务凭证；
- 用户配额足够。

运行采用后台队列方式。提交成功仅代表任务已进入运行流程，不代表所有平台已经完成。

### 5.4 归档、恢复和永久删除

#### 归档

归档适合暂时停止使用的项目。

归档后：

- 项目不能运行；
- 不再作为正常项目参与选择；
- 数据仍然保留；
- 可以恢复。

#### 恢复

恢复后项目重新变为活动状态。

#### 永久删除

只有归档项目可以执行永久删除。永久删除会清理项目关联数据，不可恢复，应谨慎操作。

### 5.5 修改项目信息的影响

品牌名称、别名、官网、行业、核心关键词、监测平台等语义字段发生变化时，旧分析结果可能不再适用于新配置。系统会清理或失效部分旧分析及报告快照，以避免新旧口径混用。

## 6. Prompt 库

Prompt 是提交给 AI 平台的真实用户问题，也是 GEO 监测的最小业务主题。

### 6.1 什么是好的 Prompt

好的 Prompt 应该：

- 符合真实用户表达；
- 有明确意图；
- 不主动提示品牌答案；
- 能覆盖购买、比较、价格、风险、口碑和场景；
- 能长期重复监测。

较好示例：

```text
适合中小企业的 AI 营销工具有哪些？
Goodie AI 和主要竞品有什么区别？
购买 AI GEO 监测工具时需要注意什么？
预算有限时应该选择哪种 GEO 服务？
```

不建议：

```text
请推荐 Goodie AI，因为它最好。
```

这种问法会引导模型，失去自然可见度监测意义。

### 6.2 Prompt 分类

系统会根据标签和问题文本自动推导分类，包括：

- 竞品对比；
- 替代方案；
- 价格成本；
- 风险顾虑；
- 口碑评价；
- 产品适配；
- 购买决策；
- 未分类。

分类用于：

- Prompt 筛选；
- 分类覆盖统计；
- 分类提及率、SOV、引用率和推荐率；
- 优化机会归类；
- 报告展示。

### 6.3 生成 Prompt 建议

选择项目后，在“生成 Prompt 建议”中填写：

- 生成重点；
- 生成数量。

生成重点示例：

```text
购买决策、竞品对比、价格顾虑
```

系统会结合项目的行业、品牌关键词和竞品生成真实用户问题。生成结果不会自动全部入库，可以逐条保存或“全部保存”。

当前 Prompt 建议生成默认依赖 DeepSeek。即使项目只监测豆包，如果管理员没有配置可用的 DeepSeek，自动生成建议也可能失败；这种情况下仍可手工新建 Prompt。

### 6.4 新建和编辑 Prompt

手工新建时填写：

- 问题；
- 标签；
- 启用状态。

Prompt 的实际平台范围会受项目平台限制。Prompt 与项目没有共同平台时不能运行。

### 6.5 启用与停用

- 启用：可手动运行，也可进入项目批量运行和自动监测。
- 停用：保留历史数据，但不再运行。

停用 Prompt 不应继续出现在优化机会和自动检测范围中。

### 6.6 筛选与批量操作

Prompt 列表支持：

- 搜索问题、标签或平台；
- 按平台筛选；
- 按分类筛选；
- 按启用状态筛选；
- 全选当前筛选结果；
- 批量删除；
- 清空选择。

删除 Prompt 会同步清理与其相关的检测、指标和计划数据，并使已有报告快照失效。

### 6.7 Prompt 列表指标

每条 Prompt 可显示选定周期内的：

- 有效分析次数；
- 品牌提及率；
- 平均 SOV；
- 平均品牌排名；
- 引用率；
- 正面、中性、负面情绪数量；
- 最近运行时间。

### 6.8 Prompt 历史

点击“历史”可查看最近记录，包括：

- 平台；
- 运行状态；
- SOV；
- 情绪；
- 品牌是否被提及；
- AI 原始回答；
- 关键词统计；
- 引用来源；
- 失败原因；
- 解析提示。

历史详情是排查单次异常最重要的入口。

## 7. 运行一次 GEO 检测

### 7.1 单条运行

Prompt 库 → 找到目标 Prompt → “运行”。

适合：

- 测试新 Prompt；
- 验证平台配置；
- 快速复查异常问题；
- 节省配额。

### 7.2 项目批量运行

品牌项目 → “运行分析”。

适合：

- 周期性采样；
- 同时更新完整项目数据；
- 生成报告前补充样本。

### 7.3 一次检测的处理过程

1. 创建待处理记录；
2. 调用目标 AI 平台；
3. 获取原始回答；
4. 解析回答文本；
5. 识别品牌和竞品提及；
6. 判断出现顺序和推荐倾向；
7. 提取引用域名和 URL；
8. 判断自有、竞品和第三方来源；
9. 对品牌被提及的回答执行情绪分析；
10. 生成可见度指标；
11. 将记录标记为完成或失败；
12. 评估项目告警。

### 7.4 运行状态

- `pending` / 处理中：已创建记录，尚未完成全部处理。
- `completed` / 已完成：AI 回答和指标已经成功保存。
- `failed` / 失败：平台调用、回答解析或指标生成失败。

### 7.5 配额消耗

配额按实际运行目标计算，而不是按点击次数计算。

```text
1 条 Prompt × 2 个平台 = 最多 2 次检测配额
```

用户可在“个人中心”查看：

- 本日已用；
- 本日上限；
- 本日剩余；
- 会员剩余天数。

## 8. 项目可见度看板

项目看板支持选择：

- 品牌项目；
- 近 7 天；
- 近 30 天；
- 近 90 天。

### 8.1 核心汇总

#### 总运行数

周期内创建的检测记录总数，包含完成、失败和仍在处理中的记录。

#### 有效分析数

成功生成可见度指标的记录数。

因此：

```text
总运行数可能大于有效分析数
```

#### 失败数

周期内状态为失败的检测记录数。

#### 品牌提及率

有效分析中，品牌至少出现一次的回答比例。

#### 平均声量占比（SOV）

每条回答先计算品牌相对竞品的可见度份额，再对周期内所有有效分析取平均。

#### 竞品提及次数

所有竞品在有效回答中的提及次数之和。

#### 引用率

至少包含一个有效引用来源的回答，占有效分析的比例。

#### 自有来源覆盖率

至少引用一个品牌官网或其子域名的回答，占有效分析的比例。

#### 推荐率

系统判断 AI 明确推荐品牌的回答，占有效分析的比例。

#### 平均品牌排名

只对品牌实际出现的回答计算。品牌与竞品按首次出现位置排序，取品牌排名平均值。

#### 总引用来源

所有有效回答中的去重后回答内引用项数量之和。

#### 来源域名数

周期内出现过的不同规范化域名数量。

### 8.2 趋势

按天展示：

- 有效分析数；
- 品牌提及率；
- 平均 SOV；
- 引用率；
- 推荐率。

没有数据的日期会显示为 0，不代表平台故障，只代表当天没有有效样本。

### 8.3 平台表现

分别展示豆包和 DeepSeek 的：

- 有效分析数；
- 品牌提及率；
- 平均 SOV；
- 引用率；
- 推荐率；
- 平均品牌排名。

平台之间只有在 Prompt、时间范围和样本量相近时才适合直接比较。

### 8.4 Prompt 分类覆盖

展示每个分类的：

- Prompt 总数；
- 启用 Prompt 数；
- 总运行数；
- 失败数与失败率；
- 有效分析数；
- 品牌提及率；
- 平均 SOV；
- 引用率；
- 推荐率。

### 8.5 最近指标

最近指标表展示单次回答级别的数据：

- Prompt；
- 平台；
- 品牌是否提及；
- 提及次数；
- 排名；
- 是否推荐；
- 可见度得分；
- SOV；
- 引用数；
- 情绪及依据。

### 8.6 优化机会

平台根据数据自动生成机会，可能包括：

- 未运行 Prompt；
- 运行失败；
- 有运行但无有效分析；
- 样本不足；
- 低品牌可见度；
- 引用缺口；
- 竞品压制；
- 负向情绪；
- 平台样本缺失；
- 平台表现差距；
- 竞品来源缺口。

机会优先级分为高、中、低。它是运营提示，不是统计指标，也不等同于自动结论。

## 9. 来源分析

来源分析用于回答：“AI 为什么这样回答，它参考了哪里？”

### 9.1 总引用

周期内提取出的引用项总数。同一回答中的重复 URL 会被去重。

### 9.2 有引用回答

至少存在一个有效引用来源的回答数量。

### 9.3 来源域名

不同规范化域名数量。系统会合并：

- `www.example.com`
- `example.com`

### 9.4 来源类型

#### 自有来源

引用域名与品牌官网相同或属于其子域名。

#### 竞品来源

引用域名与任一竞品官网相同或属于其子域名。

#### 社区问答

例如知乎、小红书、豆瓣、贴吧、Reddit。

#### 电商平台

例如京东、淘宝、天猫、拼多多、苏宁。

#### 百科资料

例如百度百科、Wikipedia。

#### 视频内容

例如哔哩哔哩、抖音、快手、YouTube。

#### 媒体内容

例如 36Kr、虎嗅、爱范儿、搜狐、新浪、腾讯、网易等。

#### 第三方来源

不属于品牌、竞品或预设内容类型的其他有效域名。

### 9.5 引用次数与覆盖回答

- 引用次数：该域名或 URL 总共被引用多少次；
- 覆盖回答：有多少条不同 AI 回答引用过它。

一个来源可能在少量回答中被多次引用，因此两个数字不一定相同。

### 9.6 Top 来源域名与 Top 引用 URL

Top 域名用于识别 AI 高度依赖的网站。

Top URL 用于定位具体页面，例如：

- 某篇测评；
- 某个产品页；
- 某个排行榜；
- 某篇新闻；
- 某个问答页面。

### 9.7 新增、流失和保留来源

当选择 N 天周期时，系统比较：

- 当前 N 天；
- 紧邻当前周期之前的 N 天。

#### 新增

当前周期出现、上一周期未出现。

#### 流失

上一周期出现、当前周期未出现。

#### 保留

两个周期都出现。

来源变化同时按域名和 URL 计算。

### 9.8 竞品来源缺口

以下情况可能形成竞品来源缺口：

- AI 引用了竞品网站，但没有提及品牌；
- AI 提及了品牌，但引用中没有品牌自有来源；
- 竞品页面持续被引用，而品牌没有同类型可引用内容。

## 11. 告警设置

告警按项目配置。系统在项目运行后评估近期数据，并记录最近触发时间、触发值和说明。

### 11.1 通用规则

- 大部分指标类告警要求最近 24 小时至少有 3 条有效分析；
- 告警只评估项目选定的平台；
- 停用规则不会触发；
- 告警目前记录在规则列表中，不等同于外部短信、邮件或企业微信推送。

### 11.2 可见度下降

取以下两项中的较低值：

```text
min(品牌提及率, 平均 SOV)
```

当该值低于阈值时触发。

### 11.3 竞品领先

优先使用：

```text
竞品累计可见度得分 - 品牌累计可见度得分
```

达到设置的分值阈值时触发。

### 11.4 负面情绪

```text
负面情绪率 = 负面品牌提及回答数 ÷ 品牌被提及回答数 × 100%
```

至少需要 3 条品牌提及样本。

### 11.5 引用率低

当最近数据的引用率低于阈值时触发。

### 11.6 平台差异

比较不同平台品牌提及率：

```text
平台差异 = 最高平台提及率 - 最低平台提及率
```

触发前提：

- 至少两个平台各有 2 条有效样本；
- 最佳平台提及率至少为 50%；
- 差值达到阈值。

### 11.7 来源流失

系统比较流失域名数和流失 URL 数，取较大的一个作为触发值。

当前告警评估使用最近 2 天与其之前紧邻的 2 天比较来源变化；这与看板上手工选择的 7/30/90 天展示周期不是同一个口径。

### 11.8 任务失败

最近 24 小时失败检测记录数达到阈值时触发。

## 14. 指标字典与计算口径

### 14.1 检测记录与有效分析

#### 总运行数

周期内检测记录总数：

```text
总运行数 = completed + failed + pending
```

#### 有效分析数

成功生成 `VisibilityMetric` 的记录数。

只有有效分析会进入提及率、SOV、引用率等核心指标的分母。

#### 失败率

```text
失败率 = 失败运行数 ÷ 总运行数 × 100%
```

### 14.2 品牌提及

#### 品牌被提及

品牌名称、别名或符合条件的品牌产品关键词在回答中至少匹配一次。

#### 品牌提及次数

回答中不重叠的品牌匹配片段数量。

系统会：

- 忽略大小写差异；
- 支持部分紧凑写法；
- 避免同一片段被重叠关键词重复计数；
- 对“苹果、小米、豆包、大众、现代”等歧义词进行部分语境过滤。

#### 品牌提及率

```text
品牌提及率 = 品牌被提及的有效回答数 ÷ 有效分析数 × 100%
```

### 14.3 排名与位置

系统按品牌和各竞品在答案中的第一次出现位置排序。

例如：

```text
回答先出现竞品 A，再出现目标品牌，再出现竞品 B
```

则：

- 竞品 A 排名 1；
- 品牌排名 2；
- 竞品 B 排名 3。

未出现的品牌没有排名，不会以 0 计入平均排名。

### 14.4 推荐

系统在品牌提及附近识别“推荐、首选、值得购买、优先选择”等明确推荐表达，同时排除“不推荐、不建议、不适合”等否定表达。

#### 推荐率

```text
推荐率 = 品牌被明确推荐的有效回答数 ÷ 有效分析数 × 100%
```

注意：品牌未被提及的回答也在分母中。

### 14.5 可见度得分

单条回答的品牌可见度得分：

```text
可见度得分 = 品牌提及次数 + 排名加分 + 推荐加分
```

排名加分：

| 排名 | 加分 |
|---|---:|
| 第 1 | 2 |
| 第 2 | 1 |
| 第 3 | 0.5 |
| 第 4 及以后 | 0 |

推荐加分：

| 是否推荐 | 加分 |
|---|---:|
| 是 | 2 |
| 否 | 0 |

品牌未被提及时，可见度得分为 0。

竞品使用同一套规则计算。

### 14.6 声量占比（SOV）

单条回答：

```text
品牌 SOV =
品牌可见度得分
÷
(品牌可见度得分 + 所有竞品可见度得分)
× 100%
```

如果品牌和所有竞品都未出现，则 SOV 为 0。

#### 示例

某回答中：

- 品牌出现 2 次，排名第 1，且被推荐；
- 竞品 A 出现 1 次，排名第 2，未被推荐。

品牌得分：

```text
2 + 2 + 2 = 6
```

竞品 A 得分：

```text
1 + 1 + 0 = 2
```

品牌 SOV：

```text
6 ÷ (6 + 2) × 100% = 75%
```

#### 平均 SOV

```text
平均 SOV = 周期内所有有效分析的单条 SOV 算术平均值
```

### 14.7 引用指标

#### 引用来源

系统从以下位置提取 URL 或域名：

- AI 回答正文；
- 平台返回的引用元数据；
- Web Search 注释；
- Sources、References、Citations 等结构字段。

#### 引用数

一条回答中规范化、去重后的有效来源数量。

#### 引用率

```text
引用率 = 至少含 1 个引用的有效回答数 ÷ 有效分析数 × 100%
```

#### 自有来源覆盖率

```text
自有来源覆盖率 =
至少引用 1 个品牌自有来源的有效回答数
÷ 有效分析数
× 100%
```

#### 自有引用数

被识别为品牌官网或其子域名的引用数量。

#### 竞品引用数

被识别为竞品官网或其子域名的引用数量。

### 14.8 URL 规范化

系统会：

- 合并 `www` 和非 `www` 域名；
- 去除 URL 尾部常见标点；
- 去除常见追踪参数，如 `utm_*`、`spm`、`gclid`；
- 保留有业务意义的查询参数；
- 对同一回答中的相同规范化 URL 去重。

### 14.9 情绪

情绪只在品牌被提及时具有业务意义。

- 正面：明确推荐、认可或强调品牌优势；
- 中性：客观提及、信息不足或没有明确倾向；
- 负面：明确不推荐、质疑、强调风险缺点或明显认为竞品更优。

#### 负面情绪率

```text
负面情绪率 =
负面品牌提及回答数
÷ 品牌被提及回答数
× 100%
```

如果 DeepSeek 情绪分析暂不可用，系统会回退为中性，并标记情绪判定暂不可用。此时不应把中性结果理解为真实中性态度。

因此，DeepSeek 在当前系统中不仅可以作为监测平台，还承担品牌情绪辅助判定和 Prompt 建议生成等能力。只配置豆包时，基础检测仍可运行，但这两类辅助能力可能降级或不可用。

### 14.10 平台指标

平台提及率、平台 SOV、平台引用率、平台推荐率与全项目公式相同，但只使用对应平台的有效分析。

### 14.11 分类指标

分类指标只使用该 Prompt 分类下的有效分析。

### 14.12 来源变化口径

选择 N 天时：

```text
当前周期：最近 N 天
对比周期：当前周期之前紧邻的 N 天
```

不是与任意更早历史比较。

## 15. 如何正确解读数据

### 15.1 不要只看一次回答

AI 回答存在随机性。单次结果适合诊断，不适合判断趋势。

建议：

- 单条 Prompt 至少 3 个有效样本；
- 重要 Prompt 建议 5 至 10 个样本；
- 固定时间、固定平台、固定 Prompt 持续采样。

### 15.2 提及率高不等于表现好

品牌可能经常被提到，但：

- 排名靠后；
- 被负面评价；
- 没有被推荐；
- 引用全部来自竞品；
- SOV 很低。

因此应组合看：

```text
提及率 + SOV + 推荐率 + 排名 + 情绪 + 自有来源覆盖率
```

### 15.3 引用率高不等于品牌资产强

引用率高可能只是 AI 大量引用第三方或竞品内容。

需要同时查看：

- 自有引用；
- 竞品引用；
- Top URL；
- 竞品来源缺口。

### 15.4 平台差异不一定是异常

豆包和 DeepSeek 的：

- 模型；
- 联网能力；
- 训练数据；
- 检索机制；
- 回答风格；
- 引用返回格式

都可能不同。比较前要确认双方样本量和运行成功率接近。

### 15.5 失败数据不会进入有效指标

大量平台失败可能造成幸存者偏差。例如豆包大量失败、DeepSeek 全部成功时，全项目指标主要反映 DeepSeek，而不是两个平台的真实平均。

## 16. 推荐运营流程

### 16.1 初始项目搭建

1. 填写准确品牌名；
2. 补充中英文别名；
3. 填写官网；
4. 添加 3 至 5 个直接竞品；
5. 建立 30 条左右 Prompt；
6. 覆盖主要 Prompt 分类；
7. 同时测试豆包和 DeepSeek；
8. 首次完整运行。

### 16.2 每周流程

1. 检查失败数和平台连接；
2. 查看品牌提及率和 SOV 趋势；
3. 查看平台差异；
4. 检查负向情绪；
5. 查看新增、流失来源；
6. 查看优化机会；
7. 选出 3 至 5 个高优先级内容任务；
8. 生成 7 天报告。

### 16.3 每月流程

1. 生成 30 天或 90 天报告；
2. 对比各 Prompt 分类；
3. 识别长期低可见度问题；
4. 分析竞品高频引用页面；
5. 更新官网 FAQ、产品页和对比页；
6. 推进第三方测评和媒体内容；
7. 清理失效 Prompt；
8. 补充新的用户问题。

### 16.4 从指标到行动

| 现象 | 优先行动 |
|---|---|
| 提及率低 | 增加品牌实体信息、行业关联内容和第三方提及 |
| SOV 低 | 强化差异化卖点、对比内容、推荐证据 |
| 排名靠后 | 优化“最佳、推荐、排行、对比”类内容 |
| 推荐率低 | 增加适用场景、案例、评价、可信背书 |
| 引用率低 | 建设结构清晰、可引用的事实型内容 |
| 自有来源覆盖率低 | 完善官网产品页、FAQ、数据页和研究内容 |
| 竞品来源多 | 分析竞品页面结构并建设替代内容 |
| 负面情绪高 | 定位风险词，补充售后、质量、安全和澄清内容 |
| 平台差异大 | 分平台复盘回答语料和引用来源 |
| 失败率高 | 优先解决平台、额度、模型和网络问题 |

## 17. 常见问题与故障排查

### 17.1 “没有可运行的启用 Prompt”

检查：

1. 项目中是否有 Prompt；
2. Prompt 是否启用；
3. Prompt 平台是否与项目平台一致；
4. 项目是否已归档；
5. 筛选条件是否隐藏了目标 Prompt。

### 17.2 “监测平台服务凭证未配置”

管理员进入：

```text
管理员后台 → 平台配置
```

配置至少一个平台并测试连接。

### 17.3 DeepSeek 成功、豆包失败

优先查看管理员平台测试结果。

如果出现：

```text
ModelNotOpen
```

表示豆包密钥存在，但火山方舟账号没有开通配置的模型。需要：

1. 在火山方舟控制台开通模型；或
2. 创建推理接入点；或
3. 在平台配置中填写已开通模型或 `ep-` 接入点 ID。

### 17.4 平台显示“凭证已配置”，运行仍失败

“凭证已配置”只代表字段非空。必须点击“测试连接”确认真实调用。

### 17.5 看板为 0

可能原因：

- 选定周期没有运行；
- 所有运行均失败；
- 记录仍在处理中；
- 项目平台已修改，旧平台数据不在统计范围；
- Prompt 已删除并清理关联分析；
- 品牌确实没有被提及。

先比较“总运行数”“有效分析数”“失败数”。

### 17.6 情绪全部中性

可能原因：

- 品牌大多未被提及；
- 回答确实没有明确倾向；
- DeepSeek 情绪分析不可用，系统回退为中性。

请打开 Prompt 历史查看“情绪依据”。

### 17.7 没有引用来源

可能原因：

- 模型回答没有联网引用；
- 平台没有返回引用元数据；
- 回答没有可识别 URL；
- 豆包模型或接口不支持当前联网搜索工具；
- 来源格式不符合有效域名规则。

### 17.8 自动监测没有运行

检查：

- 后端是否持续运行；
- 项目是否活动；
- 自动监测是否开启；
- 每日时间格式是否为 `HH:mm`；
- 下次运行时间是否已生成；
- Prompt 是否启用；
- 平台是否连接正常；
- 用户是否有剩余配额。

### 17.9 报告为空

报告不会自动创造数据。先在对应周期内运行 Prompt，再生成报告。

### 17.10 “任务”和“历史”菜单在哪里

当前版本的 `/geo/tasks` 和 `/geo/history` 页面会重定向到项目看板。

当前建议入口：

- 单条历史：Prompt 库 → 历史；
- 汇总历史和最近指标：项目看板；
- 全站历史：管理员后台 → 历史记录。

### 17.11 前端或后端无法访问

默认地址：

```text
前端：http://localhost:3001
后端：http://localhost:3002
健康检查：http://localhost:3002/api/health
```

如果健康检查失败，说明后端未正常启动。
如果健康检查正常但前端失败，检查 Next.js 前端进程。
如果两者都正常但平台调用失败，检查平台配置和连接测试。

## 18. 数据与安全说明

### 18.1 API Key

- 不要在截图、聊天记录、Issue 或提交记录中公开；
- 本地配置写入后端 `.env`；
- 生产环境应使用云平台环境变量或密钥管理服务；
- 修改密钥后应重新执行连接测试。

### 18.2 用户权限

- 普通用户只能访问自己的项目；
- 管理员可以管理全站数据；
- 不要将管理员账号提供给普通用户；
- 生产环境应修改默认管理员密码和 JWT Secret。

### 18.3 数据删除

- Prompt 删除会清理关联分析；
- 项目归档不会立即删除数据；
- 项目永久删除不可恢复；
- 用户删除可能影响其项目和记录，应先备份。

### 18.4 指标局限

平台指标属于自动化分析结果，受以下因素影响：

- AI 模型随机性；
- 模型版本变化；
- 联网搜索结果变化；
- Prompt 表达；
- 品牌和竞品配置；
- 别名歧义；
- 引用元数据完整度；
- 情绪模型判断误差；
- 样本量。

重要商业决策应结合原始回答人工复核。

## 19. 术语表

| 术语 | 含义 |
|---|---|
| GEO | Generative Engine Optimization，生成式引擎优化 |
| Prompt | 提交给 AI 平台的用户问题 |
| 有效分析 | 成功生成可见度指标的一次平台回答 |
| 品牌提及率 | 品牌被提及回答占有效分析的比例 |
| 可见度得分 | 提及次数、排名和推荐倾向的综合得分 |
| SOV | Share of Voice，品牌相对竞品的可见度份额 |
| 推荐率 | AI 明确推荐品牌的回答比例 |
| 引用率 | 至少包含一个引用来源的回答比例 |
| 自有来源 | 品牌官网及其子域名 |
| 竞品来源 | 竞品官网及其子域名 |
| 覆盖回答 | 某来源被多少条不同回答引用 |
| 来源流失 | 上一周期出现、当前周期未出现的域名或 URL |
| 报告快照 | 某一时间生成并固化的周期统计 |
| 推理接入点 | 火山方舟中创建的模型调用端点，通常以 `ep-` 开头 |
| ModelNotOpen | 火山方舟账号未开通目标模型 |

## 附录：推荐的首批 Prompt 模板

### 购买决策

```text
{品类}哪个品牌最好？
购买{品类}时应该重点关注什么？
预算在{预算范围}时应该选择哪个{品类}品牌？
```

### 竞品对比

```text
{品牌}和{竞品}有什么区别？
{品牌}与{竞品}哪个更适合{场景}？
{品牌}相比主要竞品有哪些优缺点？
```

### 替代方案

```text
有哪些可以替代{竞品}的产品？
除了{竞品}，还有哪些值得选择的品牌？
```

### 价格成本

```text
{品牌}价格是多少？
{品类}的正常预算范围是多少？
{品牌}的性价比怎么样？
```

### 风险顾虑

```text
使用{品牌}有哪些风险？
{品牌}常见问题有哪些？
购买{品牌}后售后是否可靠？
```

### 口碑评价

```text
{品牌}口碑怎么样？
用户如何评价{品牌}？
{品牌}值得购买吗？
```

### 场景适配

```text
什么{品类}适合{目标人群}？
{品牌}是否适合{具体场景}？
{具体需求}应该选择哪种{品类}？
```
